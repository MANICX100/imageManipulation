The dimensions of Bitmapped images can be manipulated by:

  • Changing the pixel density
  • Changing the pixel dimensions see the page on resolution

Changing pixel dimensions requires interpolation (calculation)

  • Measuring current pixel values
  • calculating new pixel values
  • This process is known as Resampling a digital image

This process happens all the time on an LCD TV which can only display one resolution of content known as its native resolution, all other content must be upscaled/downsampled to fit the resolution of the screen.

This was a problem with early HD Ready TVs which ran at a resolution of 1366*768 whereas content existed at 1280*720 leading to constant upscaling required.

In images there are mainly three different interpolation schemes which have varying results:

Nearest Neighbour:

  • Simplest method which adds pixels of the same colour value
  • Around the original pixel to scale the overall image
  • Color of new pixel is that of the nearest pixel of the original
  • Used by simple image editing software (magnification)
  • Can result in jagged edges and visible blocks of pixels

Bi-Linear interpolation:

  • New pixels based on an average value of 4 pixels (2^2)
  • In the adjacent 2 x 2 area of the original pixel

Bi-Cubic interpolation:

  • The most complex of the 3 interpolation schemes
  • New pixels based on a mathematical operation on 16 pixels (2^3x2)
  • In the adjacent 4 x 4 area of the original pixel
  • Takes longer to compute (requires specialist software)
  • Results in smoother edges but retains detail
